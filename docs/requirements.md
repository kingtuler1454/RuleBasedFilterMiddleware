# Аппаратные и программные требования

Данный документ описывает аппаратные и программные требования, необходимые для разработки, сборки и эксплуатации программного обеспечения **Rule-Based Filter Middleware**, расположенного в данном репозитории.

Rule-Based Filter Middleware — это middleware-библиотека для ASP.NET Core, обеспечивающая фильтрацию и валидацию HTTP-запросов на основе правил, описанных в YAML-файлах. В состав репозитория также входят демонстрационные приложения (`TestTileApi`, `TestWebApplication`) и вспомогательные скрипты (каталог `scripts`).

---

## 1. Аппаратные требования

### 1.1. Минимальные требования (для разработки и запуска тестовых приложений)

| Компонент       | Требование                                                  |
|-----------------|-------------------------------------------------------------|
| Процессор (CPU) | 64-разрядный, 2 ядра, частота от 1.8 ГГц (x64 или ARM64)    |
| Оперативная память (RAM) | 4 ГБ                                               |
| Свободное место на диске | 5 ГБ (под исходный код, .NET SDK, NuGet-пакеты и сборки) |
| Сеть            | Подключение к Интернету для загрузки NuGet-пакетов и .NET SDK |

### 1.2. Рекомендуемые требования

| Компонент       | Требование                                                  |
|-----------------|-------------------------------------------------------------|
| Процессор (CPU) | 64-разрядный, 4 ядра и более, частота от 2.5 ГГц            |
| Оперативная память (RAM) | 8 ГБ и более                                       |
| Свободное место на диске | 10 ГБ и более, желательно SSD                      |
| Сеть            | Стабильное подключение к Интернету, пропускная способность от 10 Мбит/с |

### 1.3. Требования при использовании дополнительных компонентов

Если используется хранилище запросов на базе **OpenSearch** (через пакет `OpenSearch.Client`, см. `RuleBasedFilterLibrary.csproj`), то к серверу, на котором развёрнут OpenSearch, дополнительно предъявляются следующие минимальные требования:

| Компонент       | Требование                                                  |
|-----------------|-------------------------------------------------------------|
| Процессор (CPU) | 64-разрядный, 2 ядра и более                                |
| Оперативная память (RAM) | 4 ГБ и более (рекомендовано 8 ГБ)                  |
| Свободное место на диске | 10 ГБ и более (зависит от объёма хранимых запросов) |

Для запуска ML-скриптов из каталога `scripts/ml` рекомендуется не менее 8 ГБ ОЗУ.

## 2. Программные требования

### 2.1. Поддерживаемые операционные системы

Программное обеспечение является кроссплатформенным и поддерживает все ОС, на которых работает .NET 8:

- **Windows**: Windows 10 (версия 1607 и выше), Windows 11, Windows Server 2016 / 2019 / 2022.
- **Linux**: Ubuntu 20.04 / 22.04 / 24.04, Debian 11 / 12, CentOS Stream 9, RHEL 8 / 9, Fedora, Alpine 3.17+ и другие дистрибутивы, поддерживаемые .NET 8.
- **macOS**: macOS 12 (Monterey) и новее (x64 и Apple Silicon / ARM64).

### 2.2. Среда выполнения и SDK

| Компонент             | Версия                                | Назначение |
|-----------------------|---------------------------------------|------------|
| .NET SDK              | 8.0 или совместимая                   | Сборка и разработка проекта (target framework — `net8.0`) |
| ASP.NET Core Runtime  | 8.0                                    | Запуск middleware и тестовых веб-приложений |

Целевой framework проекта — `net8.0` (см. `RuleBasedFilterLibrary.csproj`).

### 2.3. NuGet-зависимости

Зависимости основной библиотеки `RuleBasedFilterLibrary` (управляются автоматически через NuGet):

| Пакет                       | Версия    |
|-----------------------------|-----------|
| Microsoft.AspNetCore.Http   | 2.2.2     |
| OpenSearch.Client           | 1.8.0     |
| YamlDotNet                  | 16.1.3    |

### 2.4. Внешние сервисы

- **OpenSearch** (опционально) — используется в качестве хранилища истории запросов через клиент `OpenSearch.Client`. Рекомендуемая версия: **OpenSearch 2.x**. При отсутствии необходимости в анализе истории запросов сервис может не разворачиваться.

### 2.5. Инструменты разработки (рекомендуются)

- **IDE / редактор** на выбор:
  - JetBrains Rider 2023.1 и новее (в репозитории присутствует папка `.idea`);
  - Visual Studio 2022 (17.8 и новее) с поддержкой .NET 8;
  - Visual Studio Code с расширением C# Dev Kit.
- **Git** версии 2.30 и новее — для работы с репозиторием.
- **Веб-браузер** (Chrome, Firefox, Edge актуальной версии) — для работы с демонстрационной HTML-страницей `client.html`.

### 2.6. Дополнительные требования для скриптов

В каталоге `scripts` присутствуют вспомогательные скрипты (включая ML-скрипты в `scripts/ml`). Для их запуска требуется:

- **Python** версии 3.9 и новее;
- менеджер пакетов **pip**;
- Python-зависимости, указанные в соответствующих файлах внутри каталога `scripts` (при их наличии).

## 3. Сборка, запуск и проверка работоспособности

После установки .NET 8 SDK сборка решения выполняется стандартными средствами:

```bash
dotnet restore YmlRulesFileParser.sln
dotnet build YmlRulesFileParser.sln -c Release
```

Запуск тестовых приложений:

```bash
dotnet run --project TestWebApplication
dotnet run --project TestTileApi
```

Подключение библиотеки в собственный проект осуществляется через NuGet-пакет:

```bash
dotnet add package RuleBasedFilterMiddleware
```
